# DevOps: Bringing development and operations together

**Source:** https://www.atlassian.com/devops
**Section:** Related stuff

---

## Key Insights

- Elite DevOps teams deploy 208x more frequently and 106x faster than low-performing teams (DORA 2019)
- 99% of organizations report positive impact from DevOps adoption (Atlassian 2020 survey)
- DevOps lifecycle has 8 phases: Discover, Plan, Build, Test, Deploy, Operate, Observe, Continuous Feedback
- Start small: Transform a single value stream/small app rather than attempting organization-wide change
- Two toolchain approaches: all-in-one (complete but inflexible) vs open toolchain (customizable with integrations)
- Core practices stack: Continuous Integration → Continuous Delivery → Infrastructure as Code → Microservices → Monitoring
- DevSecOps integrates security into CI/CD pipeline rather than treating it as separate phase
- Culture change requires breaking silos, shared accountability across entire product lifecycle
- Over-reliance on tools fails without proper team structure and processes foundation
- Feature flags enable gradual rollouts instead of all-at-once deployments
- Microservices architecture enables independent deployment and operation of services
- "Shift left" monitoring catches issues earlier in development vs production
- Infrastructure as Code treats infrastructure definitions like actual code with version control and reviews

## Bottom Line

DevOps success requires cultural transformation and process changes first, with elite teams achieving 200x deployment frequency through automation, monitoring, and breaking down organizational silos.
